Methodology

How these brackets were generated

Heuristic brackets

Heuristic brackets are generated from the official field using seed-based win probabilities and configurable favorite thresholds for each round.

  • Each game starts from a seed-gap probability model, where better seeds are more likely to advance.
  • If the seed gap passes the configured threshold for that round, the favorite gets an extra bias.
  • A random draw decides each game, so multiple heuristic brackets can differ even with the same settings.
  • Later rounds are built from earlier winners, so every bracket is structurally consistent.

GPT brackets

GPT brackets are generated by first researching the tournament context and then asking the model to return a full bracket as strict JSON.

  • Each GPT bracket can perform its own web-enabled research pass unless a saved research file is reused.
  • The model is told to stay inside the provided field and output every round for every region.
  • If the model returns malformed or incomplete JSON, the generator retries with explicit validation feedback.
  • The result is normalized, validated, and saved in the same bracket format as the heuristic generator.
